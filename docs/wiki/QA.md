# QA 测试手册

最后更新：2026-03-07（按当前 main 分支能力校对）

## 目标

提供可重复执行的回归与基础检查清单，降低“发版后文档与真实行为不一致”的风险。

## 测试环境建议

- Android 版本：至少覆盖 Android 10、12、14
- 设备形态：手机 + 平板（或折叠屏）
- 网络：Wi-Fi / 移动网络 / 弱网
- 账号：未登录 / 已登录非大会员 / 已登录大会员

## 发布前基础检查（必测）

### 1) 播放、登录态与清晰度

步骤：
1. 使用未登录、已登录非大会员、已登录大会员三种状态分别进入视频页
2. 检查首播默认清晰度、手动切换清晰度、横屏控制区
3. 在收藏夹、稍后再看页面进入“听视频”，切换播放模式：顺序播放、随机播放、单曲循环
4. 返回首页再进入同一视频，确认续播提示与播放记忆行为正常

期望：
- 首次进入不应被错误锁死在 720P
- 播放模式、横屏控制栏与续播提示行为符合预期
- 听视频入口、封面、标题与播放内容一致

### 2) 评论区与评论交互

步骤：
1. 进入普通视频评论区与楼中楼
2. 切换评论排序（最热 / 最新）
3. 检查特殊评论标签、`UP主觉得很赞` 文案与长按可选择复制
4. 持续上下滑动评论列表，观察滚动卡顿、错位与空白复用

期望：
- 排序切换与楼中楼加载正常
- 特殊标签语义正确，复制面板可用
- 列表滑动中不出现明显掉帧、错位或重复复用

### 3) 动态、图片预览与异常恢复

步骤：
1. 在动态页快速切换多个 UP 主头像/分栏
2. 连续触发刷新与切换操作
3. 打开动态图片、评论图片和 GIF 预览，再执行关闭/返回
4. 若出现 `HTTP 412` 或加载失败，点击重试并继续切换

期望：
- 页面不出现长期空白“无数据”
- 图片预览开启与关闭自然，GIF 不异常卡死
- 异常可恢复，不会卡死在错误态

### 4) 插件中心与外部 JSON 插件

步骤：
1. 打开设置中的插件中心
2. 切换 5 个内置插件的启用状态，确认设置可保存
3. 使用一个公开可访问的 JSON 规则插件 URL 执行导入预览与安装
4. 导入后返回首页 / 播放页，确认插件状态变化不会破坏核心播放链路

期望：
- 内置插件状态切换正常
- 外部 JSON 插件可预览、导入、启停
- 插件异常不会直接破坏首页或播放主链路

### 5) 更新检查与文档一致性

步骤：
1. 在应用主入口与设置页验证更新检查入口
2. 检查更新提示弹窗、更新日志文本和跳转行为
3. 发布前核对 `app/build.gradle.kts`、`CHANGELOG.md`、`README.md` / `README_EN.md`、本 Wiki 是否同步

期望：
- 更新入口与弹窗可正常工作
- 版本号、更新日志与 README / Wiki 不出现明显错位
- 若 `versionName` 已领先 `CHANGELOG.md`，应在发布前明确补齐文档

## 回归清单（建议）

- 首页返回动画：快速返回与普通返回都应平滑
- 历史记录：批量删除与单条删除正常
- 收藏夹与稍后再看：列表滚动、点击、播放跳转稳定
- 平板 / 折叠屏：侧栏、底栏与横屏布局不破版
- 插件中心：开启 / 关闭插件后不影响核心播放链路
